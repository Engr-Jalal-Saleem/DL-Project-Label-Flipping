# DL-Project-Label-Flipping

Label Flipping Attacks in Federated Learning: URLF vs FBPLF on NSL-KDD

## Overview

This project implements and compares two label-poisoning attacks against a
federated learning (FL) system:

- **URLF** (Uniform Random Label Flipping): malicious clients flip a fraction
  of their labels to a uniformly random different class.
- **FBPLF** (Feature-Based Probabilistic Label Flipping): malicious clients
  flip labels probabilistically toward the class whose feature centroid is
  closest, producing semantically targeted (rather than random) mislabeling.

Both attacks are evaluated inside a FedAvg simulation with 50 clients training
a deep MLP (~450K parameters) on the **NSL-KDD** network intrusion detection
dataset, restricted to 3 classes: **Normal**, **DoS**, and **Probe**.

The targeted flip studied is **DoS -> Probe** — the two classes that are most
similar in feature space (both represent high-volume network traffic). This
is the tabular equivalent of a "cat -> dog" targeted flip in an image
classification setting.

## How to Run

1. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
2. Open `nslkdd_3class_deep_mlp.ipynb` in Jupyter / VS Code / Colab.
3. The notebook looks for `Datasets/KDDTrain.txt` and `Datasets/KDDTest.txt`
   first. If they're not present, it downloads NSL-KDD automatically.
4. Run all cells top to bottom.
   - `FAST_MODE = True` (20 rounds) for a quick sanity-check run (~5-10 min).
   - `FAST_MODE = False` (100 rounds) for the full results used in the report
     (~1-2 hours on CPU).
5. All figures and CSVs are saved to `results/nsl_kdd_3class/`.

## Key Results

- **Accuracy collapse vs stability**: at high poisoning intensity
  (p=1.0, N_mal=30), URLF collapses global accuracy to **6.8%** while FBPLF
  retains **84.2%**.
- **Targeted misclassification**: at p=1.0, N_mal=50, FBPLF concentrates
  **59.2%** of DoS samples into the semantically similar Probe class, versus
  only **6.9%** for URLF.
- **Attack Success Rate (ASR)**: URLF drives ASR up to **76-79%** under
  extreme poisoning, while FBPLF stays controlled at **13-22%**.
- **Detection (PoD)**: both attacks are fully detected (PoD = 1.0) by cosine
  similarity in this ~450K-parameter MLP, but FBPLF consistently shows a lower
  detection threshold than URLF, meaning its updates sit closer to the benign
  distribution.

## Dataset

[NSL-KDD](https://www.unb.ca/cic/datasets/nsl.html) — refined version of the
KDD Cup 1999 dataset (Tavallaee et al., 2009). Only the Normal, DoS, and Probe
classes are used (99.3% of the data); R2L and U2R are excluded due to extreme
class sparsity.
